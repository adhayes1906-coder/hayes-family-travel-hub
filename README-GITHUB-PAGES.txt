# Hayes Family Travel Hub — GitHub Pages Upload Folder

This package is pre-wired for the GitHub username: `adhayes1906`
Recommended repository name: `hayes-family-travel-hub`
Expected GitHub Pages URL after publishing:
`https://adhayes1906.github.io/hayes-family-travel-hub/`

Files in this folder:
- `index.html` — main app
- `manifest.json` — install/app metadata
- `sw.js` — offline cache service worker
- `icon-192.png` and `icon-512.png` — app icons
- `.nojekyll` — tells GitHub Pages not to run Jekyll

## Exact upload steps
1. Sign in to GitHub with the account `adhayes1906`.
2. Create a **new public repository** named `hayes-family-travel-hub`.
3. Open that repository.
4. Click **Add file** → **Upload files**.
5. Drag **all files in this folder** into the upload window.
6. Click **Commit changes**.
7. Open **Settings** in the repository.
8. Click **Pages** in the left sidebar.
9. Under **Build and deployment**, set **Source** to **Deploy from a branch**.
10. Under **Branch**, choose **main** and folder **/(root)**.
11. Click **Save**.
12. Wait for publishing to finish.
13. Open your live site here:
    `https://adhayes1906.github.io/hayes-family-travel-hub/`
14. On each phone, open the site and use **Add to Home Screen**.

## Updating later
- Replace `index.html` (and any other changed files), commit, and GitHub Pages will republish.

## Notes
- Keep these files together in the repository root.
- If a phone shows an older version, refresh once or remove/re-add the home-screen app so the service worker updates.
- If your GitHub username is not actually `adhayes1906`, the site URL will be different and the README will need one small edit.
